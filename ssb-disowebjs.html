what disoweb needs

Disoweb apvertrieb and all its modules must follow and cover a set of goals.

  • maintainability
  • understandable
  • extendable
  • long application lifetime

conventions for providing code

To fullfill these requirements the following standards are introduced.

  • target plattform is Firefox 3.6
    • think about memory consumption and lookups.
    • performance always matters.
    • THERE IS NO JIT COMPILER IN FF 3.6

  • test your code with jasmine
    ... don’t touch anything that doesn’t have coverage. Otherwise, you’re not refactoring; you’re just changing shit.
    – Hamlet D’Arcy
  • document you code with jsduck
  • read and use the coding conventions for writing code.
  • avoid workarounds; provide solutions. (understand what you are doing)

introduction in project structure and setup

A full featured project setup and tooling is described in confluence.

  • for IntelliJ you can find live and file templates in confluence.
  • we use IntelliJ for frontend development with hot deployment of client code.
  • tools like jsduck and jslint are provided as ready to run from confluence.
